一个新的记账软件是什么样子的？

大家可能会想到，用 AI 比以前效率高百倍。那我们就可以把所有渠道的账单，一次性全导进来。

要知道，这个“全渠道导入”，以前做起来是非常麻烦的——渠道太多，微信、支付宝、各家银行、信用卡……每一家的账单格式还都不一样，光是适配这些五花八门的文件就够喝一壶。但现在有了 AI，这件事突然变得很简单，我们几乎可以轻松把它做掉。

但这里有一个盲点：把数据导进来，其实没有任何意义。

因为导进来的只是一堆原始流水，真正的麻烦在后面——你还是得靠自己，把这些来自各个渠道的流水一条条整理清楚：给它分类，给它打 tag，甚至要回忆“这 128 块我当时到底买了啥”。而“把全渠道的数据整理清楚”，才是记账真正最大的痛点。导入解决的只是“数据在不在”，它一点都没碰“数据到底是什么意思”。

可为了让用户去整理，软件又会在导入之后加上一整套流程：分类、打标签、选账户、填备注……在电脑上都嫌烦，在手机上更是让大部分人一头雾水。大部分人其实只想记一下今天花了多少，就这么点需求，一看这么多步骤，直接就放弃了。

那现在大部分的解决方案是什么呢？是通过 AI 对话的形式，把收支语句解析成对应的数据结构，存到数据库里面。

这也是我以前写的老版本的 Flowm 的解决方案，就是我是通过导入流水的方式，给每一条流水使用 AI 进行分类，然后以这样的形式整理。但这样依旧会有个问题，很多人依旧问我，到底怎么样去导入，怎么样去配置你的 API key。

这样一来，“录”的问题好像是解决了——你不用再自己分类、打标签，动动嘴就行。但它又会引入另外一个问题：录入是省事了，可“坚持”怎么办呢？因为用这套方式，你得每天不停地对着手机说话。

只要有一次，你没有及时把数据添加完，想着今天的先累积到明天再加，然后明天又会累积到后天。人就是这样的一种生物。

但是你说你不会这样，你每次都这么操作，每天都会操作。但实际上是不会的。想象一个场景，当你在旅游的时候，你和你女朋友（或者男朋友）走在外面，买了个东西，你是买完之后接着和他聊天，还是马上记这个账？

但可能你会想到可以用快捷记账，每次聊完天，我按一下手机后面就行了。但是这个只支持苹果用户，并且在景区的情况下，网络其实不是很好，可能会卡一下，导致你的体验很不好，你就赶紧关掉走了。休息至于，你也不会整理这个记账。当你旅完游回来，你会发现累积了非常多没记的账，这个时候你又不想记了，就一直拖拖拖，拖了一个月，就没了。

所以 AI 时代，不是让记每一笔流水更方便，而是让你根本不需要去记流水——把我们各个渠道的流水直接拿出来交给 AI，让 AI 来处理。

这就是我现在设计的记账软件，不再管理录入和整理流水，而是把这些都做成接口，让 AI Agent（如 Codex、Claude Code）来根据我们提供的流水文件来做。

讲完了交互方面的设计，接下来还有另一个层面：我们记账系统本身该怎么设计？

原本我想做的是一个轻量级的复式记账，参考 Beancount 这类记账软件。但我发现了一个问题：复式记账要求非常严格的对账，每一笔的收入和支出都必须配平。

这就和前面说的录入数据强相关了——我们根本没办法很精确地录入各个渠道的数据。因为你的银行卡会和电子钱包绑定，银行卡又会和券商账户绑定，钱这么一来一回，就产生了大量重复的流水，需要不停地对账。而要想自动对账、自动更新，就得接入各个渠道的接口：要么你去对接别人的接口，要么别人主动给你提供接口——这两条路都非常麻烦，基本走不通。

所以我现在的最新设计是：干脆不再对账。每个板块的数据都彼此独立——支出就是支出，收入就是收入，资产就是资产，它们之间不做强关联。当然，它们之间可以有绑定的数据，但不会用一个数据去推导出另一个数据。

记账是一个比较私有、比较依赖于个人生活习惯风格的软件。

你的记账软件的设计，基于你个人的理念，基于你个人的生活经验，基于你生活的地方——你是美国人还是中国人，想法都是不一样的。

甚至每个人可能都有自己的一些想法，他会私信你，让你去更新。

但你完全不可能兼顾所有人的想法。

而 AI 时代，Vibe Coding 的兴起刚好能解决这个问题。

这就涉及到了 AI 时代软件的设计思考。

我们要做一个怎么样的软件才能在 AI 时代活下来？

我觉得答案是寄生在 Codex 或者 Claude Code 这种 Agent 巨型应用里面。

我们提供的是数据结构以及 UI 风格，你把基本的理念封装成一个 skill，交给你的目标用户。目标用户通过基础的软件设计，告诉 Agent 他想要一个什么样的界面。

那他每次只用和 Agent 对话，右边的 Preview 就会显示他想要的界面。因为他不懂设计，也不懂数据结构，所以我们给他提前封装好。Agent 只需要按照我们提供的设计来修改显示就行了。每次打开这个东西，只要看一下这个界面就行了。
