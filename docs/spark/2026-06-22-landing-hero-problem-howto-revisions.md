# 落地页改版：Hero / Problem / 怎么用 三处修订

- **日期:** 2026-06-22
- **范围:** `apps/web` 营销落地页
- **目标文件:** `components/Hero.tsx`、`components/overview/AppMock.tsx`、`components/Problem.tsx`、`components/Features.tsx`

## 背景

当前落地页有三处需要调整：

1. Hero 区的 App mock 套着一层「窗口」外框（macOS 红黄绿三个圆点的标题栏），且整体高度偏矮。
2. Problem（问题）区每张卡片同时给出「问题 + 示意图 + Flowm 的做法（答案）」，但答案不该出现在问题区，且示意图画的是「解法」而非「难点」。
3. Features（「怎么用」）区以 `flowm` 命令行终端为核心来讲用法，过于技术化、出现命令行；真实用法是把 GitHub 地址交给 AI Agent 代劳。

设计原则：**Problem 区只呈现痛点，解法（AI Agent / OpenAI / 口语化对话）全部落在「怎么用」区。**

---

## 1 · Hero mock：去掉窗口外框 + 加高

**文件:** `components/overview/AppMock.tsx`、`components/Hero.tsx`

- 删除 `AppMock.tsx` 中的窗口标题栏 —— 即含三个红黄绿圆点（`#e6635a` / `#e4b04a` / `#54b864`）与「Flowm — {title}」文字的那一行 `div`。
- App 预览变为无外框：内容从顶部开始，底部悬浮的 Dock 保持不变。
- 容器保留一层最小的圆角裁剪 + 柔和阴影（`rounded-[16px]` + shadow），否则白色 mock 会与页面背景糊在一起；边框可弱化或保留细线。
- `Hero.tsx` 调高 mock：
  - 桌面高度 `h-[calc((100dvh-62px)*1.3+80px)]` 的倍率 `*1.3` → `*1.55`。
  - 移动端 `max-lg:h-[800px]` → `max-lg:h-[900px]`。

**验收:** Hero mock 不再有标题栏圆点；预览区肉眼更高；Dock 切换页面照常工作。

## 2 · Problem 区：只留痛点 + 难点示意图

**文件:** `components/Problem.tsx`

每张卡片 = `tag` + 问题文案 + **难点示意图**。**整块删除**绿色竖线的「Flowm 的做法」答案区（`CARDS` 的 `answer` 字段及其渲染、`border-l-[3px] border-green` 那段）。

约束：

- 文案不出现「过去 / 现在 / 未来」。
- 不提「其他记账 app / 一堆记账 app」。
- 示意图画的是**困难**，不是解法。

### 卡片①「汇总」

- **问题文案:** 「钱散落在银行、支付宝、微信、券商……来源一大堆，格式还各不相同。」
  （主旨：数据来源很多、逐个适配是大麻烦。）
- **难点示意图（替换现有 `ReconcileViz`）:** 散落、互不相连的来源胶囊（银行 / 支付宝 / 微信 / 券商 / 信用卡），形状各异，连接线断裂或缠绕，指向一个空的「全貌？」框 —— 即它们**没能**汇到一起。
- **图注:** 「来源各异、格式不一——光凑齐就够呛」

### 卡片②「录入」

- **问题文案:** 「每天一笔笔手动记流水，又累又琐碎，没几天就坚持不下去。」
- **难点示意图（替换现有 `EntryViz`）:** 枯燥的手动录入场景 —— 重复的条目不断堆叠，且打卡早早中断（如一个迷你日历，记录停在「第 3 天」）。
- **图注:** 「一笔笔手输，没几天就断了」

> 注：原 `EntryViz` 里的 AI Agent 头像与「成功写入」对话属于解法，移至「怎么用」区（见第 3 节），Problem 区不再出现。

**验收:** 两张卡片均无答案区；示意图传达「难」；无 过去/现在/未来、无其他 app 字样。

## 3 · 「怎么用」区：步骤流程 + 对话示意图

**文件:** `components/Features.tsx`

将以命令行终端为核心的卡片组（`Terminal` / `SnapshotViz` / `BudgetViz` 三卡）替换为**「步骤 + 对话示意图」**版式。**不出现任何命令行。**

### 步骤文案（编号列表）

1. 下载并打开 Flowm 应用。
2. 打开你常用的 Agent —— **Codex、Claude Code** 或其他类似应用。
3. 把 Flowm 的 **GitHub 地址**复制过去，它就知道怎么操作。
4. 把你的流水文件（从**微信、支付宝**等导出）交给它，它帮你记进账本。
5. 要加**预算、订阅、贷款**，也都让它帮你做。

SectionHead 的 `tag` 保持「怎么用」，标题与副文案改为契合上述流程（不再说「用 flowm 命令行直接写进本地账本」）。

### 对话/流程示意图（加图）

- 风格：与 Problem 卡片一致的轻量手绘示意，置于步骤旁（如 2 栏布局：左步骤、右对话图）。
- 内容：用户把 **GitHub 地址 + 一个流水文件**发给 Agent；Agent 以 **OpenAI 头像**口语化回复，表示已帮忙记账并处理了预算 / 订阅 / 贷款。例如：
  - 用户：「这是 Flowm 的地址 + 我导出的支付宝账单，帮我记一下」
  - Agent（OpenAI 头像）：「好，6 月的 142 笔都记进账本了，预算和订阅也一并更新好啦 👍」
- **GitHub 地址用一个假链接即可**，写死 `github.com/flowm/flowm`，无需真实仓库。

**验收:** 「怎么用」区呈现 5 步流程；含一张 OpenAI 头像的口语化对话示意图；无任何命令行；产品名显示为 Codex、Claude Code。
